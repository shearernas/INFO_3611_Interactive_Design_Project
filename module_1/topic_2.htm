Topic 2: What is SQL?
What are relational databases?

SQL stands for "Structured Query Language."

It was created in the early 1970s by researchers Donald Chamberlain and Raymond Boyce while at IBM.

It is a database querying language, somewhat similar to a programming language, used to "pull" specific data from a relational database.

While it looks like a programming language, such as C or Python, SQL is not as powerful and only works within the confines of a database.

There can be no SQL without a database to use it with.

Relational databases store data in tables using categories.

Think of a table like a Microsoft Excel spreadsheet; the data exists in the cell at the intersection between the vertical rows and horizontal columns of the spreadsheet.

The rows and columns of the spreadsheet each have categories to distinguish the data, like a table, if you want to get even more simple.

These categories are what we use within SQL to query the database and "pull" the specific data we need, like using the search feature within Excel or scrolling along the spreadsheet to visually find the data you need in a specific cell.

Using this spreadsheet metaphor can help you easily understand relational tables by giving you a visual reference point.
